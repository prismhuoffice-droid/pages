subsc-receipt-bot

ChatGPT・Claude・Google AI など、月次で発生する サブスクリプション領収書 の処理を自動化する Slack ボット。
Slack に PDF を投稿するだけで「リネーム保存 → 立替払請求書 (様式第12号) 記入 → プリンタ出力 → 完了通知」までを自動実行します。
PRISM-HU が運用・提供しています。

基本機能

  • 領収書 PDF を 「投稿者_サブスク名_支払日.pdf」 規約で自動リネーム保存 (重複は _2, _3 ...)
  • 立替払請求書 (様式第12号) の 請求日 / 支払年月日 / 件名 / 金額 / 通貨 / 支払先 / チェックボックス / 申請者情報 を自動入力
  • プリンタへ 領収書 + 立替請求書 + 証拠書類 を 3 点セットで送信 (A4 1 枚に自動フィット)
  • Slack 投稿スレッドに 完了通知または エラー通知を自動返信
  • 同じ Slack メッセージ内の INVOICE + RECEIPT 等 (同支払日) は 1 書類セットに自動グルーピング
  • 同サブスクを複数人で個別契約しているケース (例: ChatGPT を担当者ごと別契約) は 担当者別 Excel ファイル で自動管理

安全機能

  • 金額変動の検知 — 前月と金額が違う場合は印刷せず警告。意図的な変動なら :white_check_mark: reaction で承認すると処理を続行
  • ヘルスチェック — 起動時にプリンタ・LibreOffice・サブスクフォルダの存在を確認、欠けていれば起動を中止
  • idempotency — 同じ投稿は 1 度しか処理しない (二重印刷防止)
  • 取り逃しの自動拾い直し — 起動時に過去 7 日のチャンネル履歴を遡って未処理投稿があれば自動処理
  • 未登録ユーザー / 解析失敗時は、印刷せずスレッドにエラー通知のみ

※ すべてのエラーは Slack スレッドに通知され、見逃しを防ぎます。

よくある質問

うちは様式第12号 (北大の立替払請求書) を使っていますが、Excel のレイアウトをカスタマイズしています。動きますか？

セル位置を設定ファイル (excel_cells) で指定する仕組みになっています。標準テンプレートからの変更が大きい場合は、セル位置をご相談いただけば対応します。完全に独自の書式を使っている場合は、テンプレート提供のうえご相談ください。

同じサブスクを複数人で個別契約しています (ChatGPT で人によってプラン違いなど)。1 つの Excel に混ざりませんか？

投稿者ごとに別 Excel ファイルを使う設定 (excel_filename = "ChatGPT_{user}立替.xlsm") があります。投稿者の名前で自動的にファイルを切り替えるので、各人のデータが混ざりません。

前月から金額が変わった場合 (値上げ・契約変更など) はどうなりますか？

前月との金額差異を検出すると、印刷せずに Slack スレッドに警告メッセージが出ます。意図的な変動なら :white_check_mark: リアクションを付けると、自動的に新金額で再処理されます。不審な変動 (誤った領収書など) はそのまま放置すれば 30 分後にキャンセル扱いになります。

常駐機材がスリープ・電源 OFF の間に投稿された PDF はどうなりますか？

起動時に Slack チャンネル履歴を遡って、未処理の投稿を自動で拾い直します (過去 7 日分)。週末に投稿された PDF も月曜朝の起動時に自動処理されます。

情報セキュリティ的な懸念はありますか？

領収書 PDF や設定ファイルはすべて教室内 (常駐機材のローカルファイル + 教室 OneDrive) に閉じています。Slack を経由するため Slack 社のサーバには投稿内容が一時保存されますが、これは Slack を業務利用していること自体と同じ範囲です。Slack トークンは設定ファイル内で権限 600 (本人のみ読み書き可) で保護されます。

新しいサブスクを追加するときは PRISM-HU に依頼が必要ですか？

教室管理者が設定ファイルに 1 ブロック追加するだけで対応できます (5 分程度)。手順は導入時のマニュアルにも記載されています。フォルダ作成も init-subsc CLI で自動化されます。

Slack をまだ使っていない教室でも使えますか？

Slack の導入支援も含めてご相談ください。事務的にはワークスペース作成 + 教室全員の招待 + 運用ルール策定 の 3 段階で、2〜3 日程度の準備が必要です。Slack 自体の使い方が他の業務 (連絡・資料共有等) でも便利になるため、サブスク処理だけでなく全般的な業務効率化に繋がります。

北大の MS 包括契約で使える Teams ではダメですか?

技術的には Teams 対応も可能ですが、現状は Slack 専用で開発しています。理由は (1) 教室管理者だけで App 導入が完結する自由度、(2) Bot 開発エコシステムの成熟度、(3) PRISM-HU 提供の他ツール (リマインド自動化・メール転送自動化) との設計統一、です。Teams は北大本部 IT 室の管理権限下にあり、Bot 導入時に学内承認が必要なケースがあるため、教室レベルで完結する Slack を採用しています。

途中で使うのをやめる場合はどうすればよいですか？

常駐機材上で launchctl unload コマンドを実行して bot を停止すれば終了です。Slack App は管理画面から削除できます。設定ファイルやログを削除する手順もマニュアルに記載しています。

ご利用にあたって

1. サービスの提供
subsc-receipt-bot は、PRISM-HU が北海道大学医学研究院内の教室・研究室向けに無償で提供する立替払請求書自動化ツールです。ソフトウェア本体およびセットアップ、初月の運用伴走サポートは PRISM-HU が無償で実施します。

2. 教室側のご負担
常駐機材 (Mac mini・Raspberry Pi 等) およびプリンタは教室にてご用意ください。クラウドストレージ (北大 OneDrive 推奨) のサブスク書類フォルダも教室側で管理いただきます。

3. データの管理
領収書 PDF・立替請求書 Excel・処理履歴 (SQLite) はすべて教室内 (常駐機材のローカルディスクおよび OneDrive) に保存されます。PRISM-HU はトラブル対応時にのみアクセスし、それ以外で教室データを参照することはありません。Slack 経由のデータは Slack 社のサーバを通過しますが、Slack を業務利用していること自体と同じ範囲です。

4. 教室管理者の責任
日常運用 (新規サブスク追加・メンバー変更・印刷確認等) は教室管理者の責任において行ってください。設定ファイルに含まれる Slack トークン等の認証情報は他者と共有しないようご注意ください。トークンが第三者に漏洩した場合は、Slack 管理画面から無効化のうえ PRISM-HU までご連絡ください。

5. 免責事項
PRISM-HU は本サービスの安定運用に努めますが、Slack・OneDrive・LibreOffice 等の外部サービスの障害・仕様変更等に起因する不具合について責任を負いかねます。重要な領収書の処理結果は、別途ご自身でも確認されることをお勧めします。

6. 利用の停止
ご利用が不要になった場合は、教室管理者にて bot を停止してください。データの削除手順はマニュアルに記載されています。PRISM-HU 側で記録している教室情報の削除が必要な場合は、別途ご連絡ください。

7. 変更について
本ガイドラインおよびソフトウェア本体は、サービスの改善に伴い変更されることがあります。変更時は本ページにて告知します。
